# Team-I OpenCLAW 实现计划

本实现目标是完整演示主方案，而不是最小可运行原型。

1. **多源数据融合**：覆盖电商订单、支付、退款、物流、评论、网关日志、设备日志、补贴台账、物流轨迹、IEEE-CIS 风格公开特征和历史案件记忆。
2. **三类业务场景**：`EC-SKIM-001` 团伙刷单骗补、`EC-FAKE-002` 空包虚假交易、`EC-ARBI-003` 补贴套利观察。
3. **模型支撑智能体**：包括风险信号智能体、模式匹配智能体、案件路由智能体、断言生成智能体、动作路由智能体、追证反思智能体、反证检索智能体、证据护照智能体和模式学习智能体。
4. **主动追证策略**：综合动作效用、预期证据增益、成本、治理风险、支持证据覆盖、反证覆盖和证据护照就绪度选择下一步动作。
5. **OpenCLAW 项目网关**：通过 `aer-audit-tools` 插件和 `/tools/invoke` 风格接口暴露受控审计动作。
6. **证据护照和人工门禁**：在案件关闭前生成可解释证据护照，并保留人工复核入口。

## 模型分配

- 路由、追证、断言、反证相关智能体：`qwen2.5-coder-7b-instruct`
- 证据护照、模式学习相关智能体：`qwen2.5-coder-14b-instruct`
- 案例记忆检索扩展：预留 Qwen3 Embedding 8B 和 Qwen3 Reranker 8B 路径。

0.5B 检查点不作为本演示的有效审计智能体模型，只能用于连通性 smoke test。

## 运行依赖

完整模型支撑运行依赖 HPC2 GPU 分配。确定性 fallback 运行用于验证系统结构；Slurm GPU 验证用于确认 LLM 真正参与完整闭环。
