# Team-I 安装与运行

本文档说明从空环境拉取仓库后，如何配置依赖、下载模型、生成数据、运行后端、启动前端和提交 GPU 验证任务。

## 目录约定

仓库采用相对路径约定，默认不依赖固定机器目录。

```text
team-i-openclaw-audit-demo/
  backend/                 # Python 后端与审计闭环代码
  data/patterns/           # 初始风险模式
  docs/                    # 中文文档与图表说明
  models/                  # 本地模型权重，默认不提交
  runtime/                 # SQLite 运行数据与中间结果
  scripts/                 # 安装、下载、运行、Slurm 脚本
```

模型权重放置在 `models/` 下，运行数据库放置在 `runtime/` 下。`models/` 被 `.gitignore` 排除，外部使用者需要在本地下载或挂载模型。

## 创建环境

```bash
git clone <repo-url>
cd team-i-openclaw-audit-demo

python -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

项目也支持可编辑安装：

```bash
python -m pip install -e .
```

复制环境变量模板：

```bash
cp .env.example .env
```

上述步骤也可以通过脚本执行：

```bash
bash scripts/bootstrap.sh
```

`.env` 中的默认模型路径为：

```text
models/Qwen2.5-Coder-7B-Instruct
models/Qwen2.5-Coder-14B-Instruct
```

## 下载模型

下载 7B 模型：

```bash
python scripts/download_models.py --model 7b --resume
```

下载 14B 模型：

```bash
python scripts/download_models.py --model 14b --resume
```

一次性下载 7B 和 14B：

```bash
python scripts/download_models.py --model all --resume
```

下载完成后，目录结构应为：

```text
models/
  Qwen2.5-Coder-7B-Instruct/
    config.json
    tokenizer.json
    *.safetensors
  Qwen2.5-Coder-14B-Instruct/
    config.json
    tokenizer.json
    *.safetensors
```

如果模型已经位于其他路径，可在 `.env` 中改写：

```text
AER_MODEL_PATH_7B=/path/to/Qwen2.5-Coder-7B-Instruct
AER_MODEL_PATH_14B=/path/to/Qwen2.5-Coder-14B-Instruct
```

## 运行确定性回归

无 GPU 环境可以运行 fallback 回归，用于验证代码、数据库、API 和前端链路。

```bash
source scripts/env.sh
AER_MODEL_BACKEND=fallback python -m aer_loop.cli run --orders 4000 --max-steps 8
python -m aer_loop.cli summary
```

该模式不调用本地大模型，输出数据库默认写入 `runtime/aer_loop.sqlite`。

## 运行本地 Transformers 模型闭环

GPU 环境中可直接使用 Transformers 加载本地模型：

```bash
source scripts/env.sh
AER_MODEL_BACKEND=local \
AER_DEMO_ORDER_COUNT=3000 \
AER_MAX_STEPS=8 \
scripts/run_model_smoke.sh
```

运行完成后会生成：

```text
runtime/aer_loop_model_smoke_<run-id>.sqlite
logs/model_smoke_run_<run-id>.json
logs/model_summary_<run-id>.json
```

## 运行 OpenAI 兼容模型服务

启动本地模型服务：

```bash
source scripts/env.sh
scripts/serve_model.sh
```

服务接口：

```text
GET  http://127.0.0.1:18088/health
GET  http://127.0.0.1:18088/v1/models
POST http://127.0.0.1:18088/v1/chat/completions
```

另开一个终端运行审计闭环：

```bash
source scripts/env.sh
AER_MODEL_BACKEND=openai \
AER_OPENAI_BASE_URL=http://127.0.0.1:18088/v1 \
scripts/run_model_smoke.sh
```

## 启动 API 与前端

使用已有验证数据库启动：

```bash
source scripts/env.sh
export AER_DB_PATH="$PWD/runtime/aer_loop_model_smoke_9728967_full_debug.sqlite"
export AER_MODEL_BACKEND=fallback
scripts/start_api.sh
```

浏览器打开：

```text
http://127.0.0.1:18081/
```

若需同时启动 API 与 OpenCLAW 网关：

```bash
source scripts/env.sh
scripts/start_demo_services.sh
```

## Slurm GPU 任务

Slurm 脚本不再绑定固定项目目录。提交前在仓库根目录执行：

```bash
sbatch scripts/slurm_model_full_debug.sbatch
```

OpenAI 兼容服务验证：

```bash
sbatch scripts/slurm_openai_model_full_debug.sbatch
```

不同集群可通过环境变量覆盖分区、模型路径和运行规模：

```bash
AER_MODEL_PATH_7B=/mnt/models/Qwen2.5-Coder-7B-Instruct \
AER_MODEL_PATH_14B=/mnt/models/Qwen2.5-Coder-14B-Instruct \
AER_DEMO_ORDER_COUNT=3000 \
AER_MAX_STEPS=8 \
sbatch scripts/slurm_model_full_debug.sbatch
```

## 数据复现说明

仓库包含最终演示所需的 SQLite 运行结果和日志摘要。外部使用者也可从初始风险模式重新生成合成业务数据：

```bash
source scripts/env.sh
AER_DB_PATH=runtime/aer_loop.sqlite \
AER_MODEL_BACKEND=fallback \
python -m aer_loop.cli run --orders 4000 --max-steps 8
```

初始风险模式位于 `data/patterns/`。运行时生成的订单、支付、退款、物流、评论、设备、IP、补贴和网关日志会写入 SQLite。

## 图表复现

实验图表可由已提交的 SQLite 运行结果重新生成：

```bash
python scripts/generate_experiment_figures.py
```

输出目录：

```text
docs/assets/experiments/data/
docs/assets/experiments/figures/
```
